About DS

A data structure is a storage that is used to store and organize data. It is a way of arranging data on a computer so that it can be accessed and updated efficiently.

  • Data structure is way of organizing data in a computer so that it can be used efficiently. Some common data structures include arrays, linked lists, stacks, queues, trees, graphs, and hash tables.
  • Different data structure excel at different operations. For example, arrays allow fast random access but can be slow for inserts/deletes. Linked lists allow efficient insertion/deletion but slow random access. Stacks/queues operate in LIFO/FIFO order. Trees and graphs allow efficient searches and spatial relationships.
  • Data structures have different time complexities for their operations. For example, accessing an array element is O(1) while searching a linked list is O(n). Knowing these complexities helps when picking optimal data structures.
  • More advanced data structures include heaps, tries, spatial data structures, self-balancing trees and so on. They have specific use cases such as priority queues, spell-checking, GIS systems, keeping data sorted, etc.
  • The implementation of the abstract data structure also matters. Factors like memory usage, caching behavior, and constant factors related to time complexity should be considered.

  • Key skills around data structure include understanding available structures and their tradeoffs, knowing when to use which structure, implementing them efficiently, and analyzing their algorithmic performance with big O notation. Mastering data structures is important for any programmer.

Classification of Data Structure:

  • Linear data structure: Data structure in which data elements are arranged sequentially or linearly, where each element is attached to its previous and next adjacent elements.
    Examples of linear data structures are
    array, stack, queue, linked list, etc.
  • Static data structure: Static data structure has a fixed memory size. It is easier to access the elements in a static data structure.
    -An example of this data structure is an array.
  • Dynamic data structure: In dynamic data structure, the size is not fixed. It can be randomly updated during the runtime which may be considered efficient concerning the memory (space) complexity of the code.
    -Examples of this data structure are queue, stack, etc.
  • Non-linear data structure: Data structures where data elements are not placed sequentially or linearly are called non-linear data structures. In a non-linear data structure, we canâ€™t traverse all the elements in a single run only.
    -Examples of non-linear data structures are trees and graphs.

Queue

A queue is a linear data structure that follows the first-in, first out (FIFO) principle. The key features of a queue are:

    Insertion happens at the rear end of the queue. This operation is called enqueue.
    Deletion happens at the front end of the queue. This operation is called dequeue.
    The oldest element inserted in the queue is processed first. Hence queues follow the FIFO order - First In First Out.

Some key properties of queues:

    New elements are added at the back/rear of the queue.
    Elements are removed from the front of the queue.
    Front points to the front/first element of the queue. Rear points to the last element.
    Queue operations are enqueue (insert) and dequeue (remove).

    Stack

    A stack is another fundamental linear data structure that follows the last-in, first-out (LIFO) principle. The main operations of stack are:

      Push - Adds an element to the top of the stack.
      Pop - Removes the top element of the stack.

    Some key properties of stacks:

      New elements are added/pushed at the top of the stack.
      Last inserted element is removed first. Hence it follows LIFO order - Last In First Out.
      Top points to the last inserted element.
      Main operations are push and pop.

      Linked List

      A linked list is a linear data structure where each element is a separate object that contains a pointer or reference to the next element. Unlike arrays, elements in a linked list are not stored contiguously in memory. - The main operations associated with linked list include:

        Insert - Inserts a new node at the beginning, end or middle
        Delete - Deletes a node by key/value
        Traverse - Print out the linked list
        Search - Search for a value in the linked list
        Sort - Sort the linked list

      Some key properties of linked lists:

        Each element is called a node. It contains data and a pointer/reference to the next node.
        The first node is called the head. The last node points to null indicating end.
        Insertion and deletion of nodes does not require reallocation of the entire structure.
        operations like insert, delete, traverse can be done efficiently in O(1) time. But search is O(n) since elements not indexed.
